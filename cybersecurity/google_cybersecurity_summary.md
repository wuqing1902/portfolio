## Module 1
This section focused on building a strong foundation in core cybersecurity principles, common threats, and key security roles. The CIA triad (Confidentiality, Integrity, Availability) was applied to evaluate security scenarios and understand how systems maintain data protection and operational reliability. Practical exercises involved identifying and analyzing common attack techniques such as phishing, malware, and social engineering, with emphasis on recognizing attacker behavior and potential indicators of compromise. In addition, SIEM dashboards were used in lab environments to monitor and interpret simulated security events, helping bridge the gap between theoretical concepts and real-world security operations. Overall, this work developed a foundational understanding of cybersecurity fundamentals, threat types, and the broader threat landscape, providing a base for more advanced security analysis and operational tasks.

## Module 2
This section focused on understanding key cybersecurity domains, frameworks, and security controls. The NIST Risk Management Framework (RMF) was applied in structured scenarios to understand how organizations identify, assess, and manage security risks in a systematic manner. Different types of security controls—technical, administrative, and physical—were evaluated to understand how layered defenses work together to reduce risk and protect assets. In addition, foundational Identity and Access Management (IAM) concepts such as access control and the principle of least privilege were practiced to understand how organizations restrict and manage user permissions effectively. Overall, this work developed the ability to assess cybersecurity risks and understand how structured frameworks and controls are used by organizations to manage and enforce security.

<br><br><br><br><br>





## Module 3
This section focused on understanding network architecture, protocols, and techniques for detecting malicious activity. Network traffic and packet structures were analyzed using tools such as Wireshark and tcpdump to understand how data flows across systems and how different protocol fields contribute to communication behavior. Common network-based attacks, including SYN floods and ICMP-related anomalies, were identified through traffic inspection and pattern recognition, helping build awareness of how abnormal traffic patterns indicate potential security threats. In addition, foundational network hardening concepts such as segmentation, firewall rule implementation, and multi-factor authentication were applied to understand how layered defenses reduce exposure to attacks. Overall, this work developed practical skills in network traffic analysis and strengthened the ability to identify and interpret suspicious network behavior in real-world scenarios.

<br><br><br><br><br>





## Module 4
This section focused on system-level investigation and data analysis using core Linux and SQL tools. Linux command-line utilities such as grep, find, chmod, and user management commands were used to analyze system logs, locate relevant files, and manage access permissions, reinforcing an understanding of how system activity can be monitored and controlled at the operating system level. File permission modifications and user management practices were applied to explore access control concepts and enforce basic security principles. In addition, SQL queries were written to filter and investigate login activity, enabling structured analysis of authentication data and suspicious behavior patterns. Overall, this work built a strong foundation in system investigation, log analysis, and structured data querying, supporting the ability to identify and interpret security-relevant events at the system level.

<br><br><br><br><br>





## Module 5
This section focused on risk assessment and vulnerability management within organizational security contexts. Assets were classified based on sensitivity levels ranging from Restricted to Public, helping to prioritize protection based on data criticality and potential business impact. Risk assessments were developed using the likelihood × impact model to evaluate and prioritize security risks in a structured and consistent manner. In addition, basic threat modeling and vulnerability identification exercises were conducted to understand how weaknesses can be discovered and assessed before exploitation. Overall, this work developed a risk-based mindset and strengthened understanding of how organizations prioritize security efforts based on asset value, threat likelihood, and potential impact.

<br><br><br><br><br>





## Module 6
This section focused on Security Operations Center (SOC) workflows and incident handling processes. The incident response lifecycle was practiced end-to-end, including detection, analysis, containment, and recovery, to understand how security incidents are managed in a structured and time-sensitive manner. Alerts were investigated using SIEM and IDS tools to correlate logs, validate suspicious activity, and identify potential security threats. Findings were documented in a structured format to ensure traceability, consistency, and alignment with professional incident response practices. Overall, this work developed foundational skills in security monitoring, alert triage, and incident response workflows, strengthening the ability to respond effectively to security events in a SOC environment.

<br><br><br><br><br>





## Module 7
This section focused on automation and data processing using Python to improve efficiency in security-related workflows. Python scripts were developed to parse log files and extract relevant security data, enabling faster and more structured analysis of large datasets. File handling techniques such as reading and writing files were applied alongside regular expressions (regex) to identify patterns, filter information, and process unstructured log content. In addition, simple security tasks were automated to reduce manual effort and improve consistency in repetitive analysis processes. Overall, this work enhanced the ability to apply Python for automation in cybersecurity contexts, particularly in log processing, pattern detection, and streamlining routine investigative tasks.

<br><br><br><br><br>





## Module 8
This section focused on communication, incident escalation, and overall career readiness in cybersecurity operations. Incident escalation and prioritization were practiced based on business impact and severity, ensuring that security events were routed appropriately according to urgency and organizational requirements. Structured reporting was developed for both technical and non-technical audiences, translating detailed security findings into clear and meaningful insights for different stakeholders. In addition, AI tools were used to assist with documentation and analysis, improving efficiency while maintaining accuracy through validation of outputs. Overall, this work strengthened professional communication skills and readiness for SOC environments by improving the ability to prioritize incidents, present findings effectively, and support decision-making across technical and business teams.

## Summary
This portfolio work demonstrates hands-on application of core cybersecurity concepts across multiple domains in lab-based environments. The CIA triad, risk frameworks, and security controls were applied to evaluate and strengthen security scenarios, supporting structured decision-making around confidentiality, integrity, and availability. Network analysis was conducted using Wireshark and tcpdump to inspect packet structures and identify anomalies and potential attack patterns, reinforcing understanding of real-world network threats. SIEM and IDS tools were used to monitor, detect, and investigate security events by correlating logs and validating alerts. In addition, asset classification, risk assessment, and basic threat modeling were performed to prioritize security efforts based on business impact and potential vulnerabilities. System-level investigation was supported through Linux CLI and SQL, enabling structured log analysis, access control review, and authentication data investigation. Automation was introduced using Python scripts to parse logs and streamline repetitive security tasks, improving efficiency in data processing and analysis. Finally, incident response playbooks and SOC workflows were practiced to simulate structured detection, escalation, containment, and response procedures. Overall, this work builds a strong foundation in practical cybersecurity operations, combining analysis, automation, and incident handling within realistic security scenarios.
